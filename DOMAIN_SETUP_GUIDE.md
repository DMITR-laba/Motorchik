# Руководство по настройке домена

## Настройка домена app.domain для локального использования

### Windows

1. **Откройте файл hosts с правами администратора:**
   - Нажмите `Win + R`
   - Введите: `notepad C:\Windows\System32\drivers\etc\hosts`
   - Нажмите `Enter`
   - Если появится запрос UAC, нажмите "Да"

2. **Добавьте запись в конец файла:**
   ```
   127.0.0.1    app.domain
   ```

3. **Сохраните файл** (Ctrl+S)

4. **Очистите DNS кэш:**
   - Откройте командную строку от имени администратора
   - Выполните: `ipconfig /flushdns`

### Linux/Mac

1. **Откройте файл hosts:**
   ```bash
   sudo nano /etc/hosts
   ```
   или
   ```bash
   sudo vi /etc/hosts
   ```

2. **Добавьте запись в конец файла:**
   ```
   127.0.0.1    app.domain
   ```

3. **Сохраните файл:**
   - В nano: `Ctrl+O`, затем `Enter`, затем `Ctrl+X`
   - В vi: `:wq` и `Enter`

## Перезагрузка Nginx после изменения домена

После сохранения домена в интерфейсе, конфигурация Nginx обновляется автоматически, но для применения изменений нужно перезагрузить Nginx:

```bash
docker-compose exec nginx nginx -s reload
```

Или перезапустите контейнер Nginx:

```bash
docker-compose restart nginx
```

## Проверка работы

1. Убедитесь, что запись добавлена в hosts файл
2. Откройте браузер и перейдите на `https://app.domain`
3. Примите самоподписанный SSL сертификат (если используется HTTPS)
4. Должен открыться интерфейс приложения

## Для продакшена

Для реального домена:

1. Настройте DNS запись типа A, указывающую на IP вашего сервера
2. Убедитесь, что домен разрешается в DNS (проверьте через `nslookup` или `dig`)
3. Получите SSL сертификат через Let's Encrypt (см. `NGINX_SSL_SETUP.md`)
4. Сохраните домен в интерфейсе
5. Перезагрузите Nginx
